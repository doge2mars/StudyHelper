## 🤖 AI 智能录入说明 (V2.2+ 新特性)
Study Helper Pro V2.2 引入了强大的「AI 智能解析」快捷导入功能。你可以直接将 AI 生成的纯文本题目内容粘贴到系统中，系统会自动帮你填入文本、选项、答案和解析，无需手动一次次复制粘贴。

### 推荐的 AI 提示词与文本格式配置
在与 ChatGPT、Kimi、DeepSeek 等 AI 对话时，请让 AI 输出带有以下特定标签的格式：

```text
[题干] 设集合 A={1,2}，则下列表述正确的是...
[A] 1
[B] 2
[C] 3
[D] 4
[答案] A
[类型] 单选题
[解析] 根据集合的定义，我们可以得出...
```

**支持被识别的标签（任选其一，支持方括号或中文【】）：**
- **题干**: `[题干]`, `[题目]`, `[问题]`
- **选项**: `[A]`, `[B]`, `[C]`, `[D]` (或者 `[选项A]` 等)
- **答案**: `[答案]`, `[正确答案]`, `[标准答案]`
- **解析**: `[解析]`, `[解答]`, `[讲解]`
- **类型 (可选)**: `[题型]`, `[类型]` (支持单选、多选、填空、主观)

## 🛠️ 纯净重置步骤
如果你想抹除所有测试数据，重新经历一次完整的安装流程，请在 NAS 终端中依次执行以下命令：

### 1. 停止当前运行的旧容器
```bash
cd /vol2/1000/work/test/StudyHelper_V14
docker compose down
```

### 2. 清理旧数据（⚠️ 危险操作：确认无保留必要后再执行）
删除原来的 SQLite 数据库文件：
```bash
rm -rf data/study_pro.db
```

清空之前测试上传的题目图片、解析图和视频文件：
```bash
rm -rf static/uploads/* 
```

### 3. 重新构建并启动全新系统
使用本地（刚刚从 Mac 同步过来的）最新 V2.3.0 源码构建 Docker 镜像，并在后台强行启动：
```bash
docker compose up -d --build
```
